C. Hybrid

Recommended starting point: napari assembles/opens the meta database; the browser handles exploration. The backend stays pure Python so CLI, tests, and future UIs share one contract.

Likely invariant

  • Build a stable meta database first, independent of UI.
  • Represent cells, tracks, frames, positions, conditions, metrics, events, and analysis results as queryable tables.
  • Keep analyses as backend modules that consume filtered tables and emit tables plus figures.

Main uncertainty

  • Whether users need image/label-linked inspection during meta analysis, or mostly aggregate experiment-level exploration.
  • How much SQL power to expose directly versus through guided grouping/filter controls.
  • Whether the first product should prioritize one experiment at a time or cross-project aggregation.
