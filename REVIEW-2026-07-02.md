## Summary

The prior review was thorough and its fixes hold up. On independent inspection,
MSD, diffusion/persistence, order parameter, circularity, IoU, divergence
sign/axis, and ICM cost are all correct. **No new critical bugs.** The new
findings are one confirmed scientific-correctness bug, two confirmed latent
bugs, a batch of genuinely dead code, and a default-config performance no-op.

## Confirmed bugs

### 1. Medium (scientific) — cell–cell contacts made of ≥2 non-adjacent boundary pixels are silently dropped
`aggregate_quantification/contacts/build.py:675` (`_coordinate_segments`).

`_coordinate_segments` returns `[coords]` for a single boundary-crossing point,
but returns `[]` when there are two or more crossings that aren't adjacent to
each other — the `while unused_edges` loop never runs. Reproduced firsthand:

```
frame = [[1,2],
         [1,3],
         [1,3],
         [1,3],
         [1,2]]        # cells 1 and 2 touch at rows 0 and 4 only
→ _extract_frame_cell_edges emits an edge row for pair (1,3) but NONE for (1,2)
```

Consequences on the canonical artifact:
- `cell_neighbor_counts` under-reports (1 and 2 not counted as neighbors that
  frame).
- `label_contacts` omits the (1,2) contact row.
- `signed_central_junction_lengths` drops samples in exactly the
  fragmented/near-four-fold-vertex states the reaction coordinate is most
  sensitive to.

Fix: emit each orphan crossing as its own length-0 (or length-1) segment,
mirroring the single-point (`len(coords) <= 1`) branch.

### 2. Medium (latent) — grouped `Collapse` crashes on an all-parseable object-dtype column
`aggregate_quantification/reduce.py:215` (`Collapse.apply`).

The no-`by` branch coerces each value column with
`pd.to_numeric(df[c], errors="coerce")` (line 207), but the grouped branch calls
`grouped[numeric_values].agg(agg)` on the raw dtype. `_is_numeric` deliberately
treats an object column whose values all parse as numeric (e.g. numeric strings)
as a value column, so the same spec succeeds ungrouped and raises
`TypeError: agg function failed [how->mean, dtype->object]` grouped.

Latent because the in-scope CSV readers (`read_table_csv`) coerce value columns
to `float`; it bites `reduce` as the generic table→table layer it advertises
itself to be. Fix: coerce with `to_numeric` in the grouped branch too (mirror
line 207).

### 3. Medium (latent) — post-solve collision pass can destroy an anchored track under an id coincidence
`tracking_ultrack/corrections.py:1174` (`apply_post_solve_corrections`).

The reserved-id collision loop relabels `labels == cell_id` to a fresh id
without excluding pixels of a solver track that is about to be remapped *into*
that value. The parallel code in `reseed.py:400` guards this with `& ~src_mask`;
this path is missing it.

Scenario: anchors 10 and 20. The solver assigns track id `20` to the track that
anchor 10 matched, and `30` to anchor 20's track, giving
`solver_track_remap = {20:10, 30:20}` and
`matched_anchor_target_to_solver = {10:20, 20:30}`. The collision loop for
`cell_id=20` relabels `labels==20` (solver track 20, destined to become 10) to a
fresh id; the later `20→10` remap then finds nothing, so anchored cell 10 ends
up with a random fresh id instead of its guaranteed identity. Requires a numeric
coincidence between a solver-assigned track id and a *different* anchored cell
id — which is exactly the collision case this code exists to handle.

Verified by tracing + contrast with `reseed.py`, not executed (needs a full DB
fixture). Fix: exclude remap-source pixels before relabeling, as `reseed.py`
does.

## Complexity / performance

- **Default config runs an expensive filter that multiplies by zero.**
  `core/imageops.py:13 residual` always computes
  `threshold_local(method="gaussian")` even when `strength=0`, and
  `CellDivergenceParams.fg_strength` (`cell_divergence_segmentation.py:63`)
  defaults to `0.0`. Every default-config frame (including live previews) pays
  an O(T·Y·X) gaussian for a no-op. Guard:
  `if strength == 0: return np.clip(frame, 0.0, None)`.
- **Redundant re-ordering in contacts.** `build.py:346,357` call
  `_order_coordinates(segment)` on segments that `_coordinate_segments` already
  returns in traversal order — the endpoint-finding + greedy walk runs twice.
  Pure duplicated work, no correctness impact.
- **Unary cache key omits input content.** Beyond being dead,
  `_unary_cache_key` (`cell_label_icm.py:438`) hashes only `T×Y×X_α_γ`, not
  `contours` / `nuc_tracks` / `fg_mask` / `foreground_scores`. If anyone ever
  passes `cache_dir`, changing an upstream knob that alters the cost field but
  not α/γ silently returns a stale segmentation. Fix the key before this path is
  wired up.

## Statistical caveats (Low — correctness-adjacent, not bugs)

- **MSD / DAC SEM assumes independent samples.** `msd.py:70` and
  `kinematics.py` report `SEM = std(ddof=1)/√N` over overlapping-origin samples
  that are strongly autocorrelated, so `N` overcounts the independent-sample
  count and the persisted error bars are systematically too small at large lags.
  Estimated means (D/α/persistence) are unaffected.
- **`_default_n_max` uses frame span, not effective length.**
  `msd.py:149` — one gappy 2-point track (present only in frames `{0, 400}`)
  drives `n_max` to 100, extending the ensemble MSD curve into an unreliable,
  poorly-sampled tail.
- **`_one_over_e_length` can return a bin center instead of NaN.**
  `collective.py:164` — when the pooled `C(r)` is already ≤ 1/e at the first
  populated bin, it returns that bin's center as ξ rather than the honest
  "cannot resolve" NaN.

## Verified correct (to bound the review)

- MSD (`D = e^intercept/(2d)`, d=2), diffusion/velocity units, DAC persistence
  (`P = -1/slope`), order parameter, circularity, orientation folding,
  pixel/frame-interval unit conversions.
- Neighbor-count adjacency (distinct-neighbor sets), cell density (div-by-zero
  guarded, cells/mm²), signed-length fragment join + sign convention, T1
  topology test, catalog identity dedup, shape-table joins.
- IoU / intersection cropping, centroid-corrected IoU rasterization,
  centroid-distance prefilter radius, seed-prior math, atom merge tree,
  node-id arithmetic, frame off-by-ones — all coordinate-consistent (yx
  throughout).
- Divergence sign/axis (`div = ∂dy/∂y + ∂dx/∂x`), flow-following axis ordering,
  ICM cost field + streaming argmin + `_INF` masking, label dtype handling.
